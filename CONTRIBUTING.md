# Contributing to optirule

Thanks for your interest in improving optirule! Contributions of all kinds are
welcome — bug reports, new agent adapters, and docs fixes.

The full contributor guide lives in the README so it stays next to the project
docs:

- **[Development setup](README.md#development)** — install, build, test, typecheck.
- **[Contributing guidelines](README.md#contributing)** — how to file issues, the
  pull-request checklist, and how to add a new agent adapter.

## The short version

1. Open an [issue](https://github.com/BaconMan1168/optirule/issues) first for
   anything non-trivial, so we can agree on the approach before you write code.
2. Fork, branch off `main`, and make one focused change per PR.
3. Add or update tests for any behavior you change.
4. Make sure `npm run typecheck` and `npm test` both pass before pushing.
5. Use [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`,
   `fix:`, `docs:`, …) — it's what the project's history uses.

By contributing, you agree that your contributions will be licensed under the
project's [MIT License](LICENSE).

Please also review our [Code of Conduct](CODE_OF_CONDUCT.md).
